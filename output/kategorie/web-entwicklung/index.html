doNotTrack Browserübergreifend erkennen

Mit dem doNotTrack Befehl können User si­gna­li­sie­ren, dass sie nicht von Websites getrackt werden möchten. Browser senden dafür einen HTTP-Header mit ihren Requests an den Webserver. Via JavaScript lässt sich diese Ein­stel­lung auch im Client auslesen. Dieses Auslesen funk­tio­niert leider nicht ein­heit­lich.

Chrome und Safari haben eine andere Schreib­wei­se für die doNotTrack Property als der Firefox Browser. Der IE 9/10 verwendet einen eigenen Namen. Und der IE 8 un­ter­stützt diese Ein­stel­lung laut Microsoft auf sehr krude Art und Weise.

Ein Tipp für die Leute mit eigener Domain: Wei­ter­lei­tung von tho­ma­s­pup­pe.de/+ auf das Google-Profil via .htaccess:
